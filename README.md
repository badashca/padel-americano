# 🎾 Americano · Падел-турнир

Минималистичный планировщик турнира по паделу в формате **Американо** — без бэкенда, работает прямо в браузере, идеально подходит для GitHub Pages.

## Что умеет

- **Умный рандом**: алгоритм минимизирует повторы напарников и оппонентов, справедливо распределяет отдых
- **Авторасчёт формата**: по числу игроков, кортов и доступному времени подсказывает оптимальное число раундов и очков на матч
- **Live-таблица**: очки и победы по каждому игроку обновляются после ввода счёта
- **Сохранение**: всё хранится в `localStorage`, можно закрыть вкладку и продолжить
- **Mobile-first**: рассчитано на использование с телефона прямо у корта
- **Поддержка 6+ игроков и нескольких кортов одновременно**

## Деплой на GitHub Pages

1. Создай новый репозиторий, например `padel-americano`
2. Залей файл `index.html` в корень репозитория:
   ```bash
   git init
   git add index.html README.md
   git commit -m "init"
   git branch -M main
   git remote add origin https://github.com/<твой-логин>/padel-americano.git
   git push -u origin main
   ```
3. В репозитории → **Settings** → **Pages** → Source: `Deploy from branch` → Branch: `main` / `/ (root)` → **Save**
4. Через минуту сайт будет доступен по адресу:
   `https://<твой-логин>.github.io/padel-americano/`

## Как играть

1. **Шаг 1 — Настройки.** Выбери число игроков, кортов и время. Приложение само подберёт раунды и очки на матч (можно подправить вручную).
2. **Шаг 2 — Состав.** Добавь имена. Минимум 4, рекомендуется 6+.
3. **Турнир.** На каждый раунд автоматически сгенерируются пары. Вводи счёт первой команды — вторая считается автоматически (Американо играется до фиксированной суммы очков).
4. **Таблица.** В любой момент жми вкладку «Таблица», чтобы посмотреть, кто лидирует.

## Формат Американо (краткое напоминание)

- Каждый матч играется до **фиксированной суммы очков** (например, 24): победители 16, проигравшие 8, или 13:11, итог всегда = 24
- **Каждый игрок копит очки лично**, а не команда — состав команд меняется каждый раунд
- Побеждает тот, у кого больше всего очков в сумме по всем сыгранным раундам

## Технологии

Чистый HTML/CSS/JS, без сборки и зависимостей. Один файл. Шрифты — через Google Fonts (Bricolage Grotesque, Outfit, JetBrains Mono).
